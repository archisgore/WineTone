# WineTone v2026.05.21 — Auth, Submissions, and a Fine-Tuned Encoder

*Released May 21, 2026.*

Three weeks ago WineTone was a 6-label demo with a single hand-typed
username. Today it has real authentication, lets anyone add wines to
the catalog, talks back to you in plain English, and runs on a wine-
corpus-tuned encoder. This is the release where it stops feeling like
a research script and starts feeling like a product.

If you've been calibrating on
[tone.wine](https://tone.wine) before today, your old `archis`-style
pseudonymous account is gone — auth is now required to label wines.
Sign in once, re-label a handful, and you're back. The whole calibrate
flow takes ~five minutes.

---

## What's new for users

### Sign in with email, Google, or GitHub

Auth via [Clerk](https://clerk.com). Every label is now tied to a real
account; usernames are unique and enforced by the auth provider. Read
access stays anonymous — anyone can browse anyone's labels, run
ad-hoc queries at `/ask`, or search the public vocabulary corpus at
`/vocab`. Only *writing* requires sign-in.

The privacy banner remains: this is still a research prototype and
everything you label is public. Pick a pseudonymous Clerk username if
you want anonymity.

### Add wines to the catalog

The 164,069-wine corpus came from public review sources, and inevitably
has gaps — small producers, recent vintages, wines from outside the
WineEnthusiast review universe. There's now a "Add a wine" link in the
header. Type producer, vintage, variety, country/region, and a free-text
description, and your wine joins the catalog immediately. Idempotent:
re-submitting the same (producer, wine, vintage) returns the existing
entry. Sources are tracked, and user-submitted wines are searchable to
everyone within seconds.

### Ask in your own words

The new `/ask` page accepts free-form natural language. It runs through
an LLM router (Llama-3.1-8B via HF Inference, free with HF Pro credits)
that decides which backend to invoke and produces a comparison table
or narrative when you ask for one.

Worked examples:

- **"Find me a cheaper alternative to Caymus Special Selection."** The
  router parses out `intent=alternative_to, reference=Caymus, max_price=80`
  and shows you six Napa Cabs at cosine ≥ 0.94 to Caymus for $40-75.
  The same wine in taste-space at 60% off.
- **"Two Buck Chuck."** Routes to a recommend with `max_price=10`.
  Catalog doesn't have Charles Shaw itself — but cheap drinkable
  alternatives at the same price tier.
- **"Show me wines someone described as sunshine in a bottle."** Routes
  to vocabulary search — returns the Tokaj Aszú bottles that someone
  on the platform labeled with that exact phrase.
- **"Compare Caymus to cheaper alternatives in the same flavor class,
  with a column for flavor distance."** Builds the table inline, with
  `flavor_distance = 1 - cosine` as the third column.

The conversational narrator only uses data actually present in the
results — no hallucinated descriptions, prices, or wines.

### Vocabulary search

The corpus of *user-written tasting notes* is now its own search
index. At `/vocab`, type any phrase and find the wines someone has
described that way. "Theatre popcorn butter" finds the Louis Roederer
Cristal a user labeled with that phrase. "Volcanic minerals jasmine"
lands on Santorini Assyrtiko. This is the "your oaky isn't my oaky"
thesis as a directly searchable feature — the public catalog of
personal vocabularies is itself useful.

### Custom domain on tone.wine

Live at https://tone.wine via HF Pro. The site is *exactly* what
ships from the GitHub repo, no separate frontend codebase.

## What's new under the hood

### Fine-tuned encoder: `archisgore/bge-small-winetone`

The off-the-shelf bge-small-en-v1.5 encoder was trained on general web
text. Wine-specific terms — "petrol," "graphite," "stewed fruit," "VA"
— landed near their literal English meanings, not their wine-reviewer
meanings.

Today it's been replaced with a fine-tune contrastively trained on
**173,708 review pairs from the WineTone corpus** — for each canonical
wine with ≥2 reviews, every pair of its reviews became a positive
training example, with in-batch other wines as implicit negatives.
1 epoch, MultipleNegativesRankingLoss, ~28 min on Apple Silicon MPS.

The full 164,069-wine corpus has been re-encoded against the new model.
Same 384-dim space, L2-normalized — downstream calibration and cosine
scoring are unchanged. Just better.

Available on Hugging Face Hub at
[archisgore/bge-small-winetone](https://huggingface.co/archisgore/bge-small-winetone)
for anyone who wants the model standalone. Apache-2.0.

### Negative-label calibration

The personal projection now distinguishes "wines I want more of" from
"wines I described but don't want more of." Add a label with thumbs-up
or thumbs-down; positives pull the projection toward the wine,
negatives push it away (hinge loss with `margin² = 2.0` — cosine ≤ 0
contributes zero loss). End-to-end verified: a three-label fit with
one positive Brunello and two negative Napa Cabs pushes the negatives
to cosine -0.27 against the user's negative vocabulary.

### Sparse channel: Postgres FTS instead of joblib TF-IDF

The previous lexical channel was a scipy.sparse TF-IDF matrix in a
joblib file. Couldn't grow incrementally — any wine added after the
joblib was built got dense-only scoring forever. And the ephemeral
container storage on HF Spaces meant we couldn't durably mutate it.

The new lexical channel is a `tsvector` column on the `wines` table
with a GIN index. INSERT triggers update it instantly; user-submitted
wines are lexically searchable the moment the submission commits.
Persists with the DB, no file dependency. The `wines.tsv` includes
producer/wine/variety/region/country **plus** the full review text
folded in via `to_tsvector('english', ...)`.

### Price-aware queries

`/recommend` and `/ask` both honor optional `max_price` and `min_price`.
The LLM router understands named price tiers — "Two Buck Chuck" maps
to `max_price: 10`, "splurge" to `min_price: 75`, "Tuesday wine" to
`max_price: 30`. The CLI gets `--max-price` and `--min-price` flags.

92.8% of catalog wines have prices (from WineEnthusiast review-time
list prices). Median wine in the catalog is $25, p90 is $65, max is
the $3,300 Château les Ormes Sorbet 2013.

### Find-cheaper-alternative-to-X

The killer arbitrage feature. Given any wine in the corpus, return the
k nearest neighbors in embedding space filtered by price. Demo: Caymus
Special Selection ($130) → top alternatives are all Napa Cabs at $40-75,
cosine ≥ 0.944, 63-69% savings. The dense embedding already does the
flavor-clustering work; this just sorts the cluster by price.

CLI: `winetone alternatives "<wine>" --max-price 100`. Web: routed
automatically via `/ask` when you mention a wine and a budget.

## Behind-the-scenes operational changes

- **Hosting moved to HF Pro Spaces + Neon Postgres.** ~$9/mo for the
  Space, free Neon tier sufficed until the FTS rebuild pushed us
  through the 512 MB cap; now on the paid Neon plan. Total recurring
  cost ~$25/mo.
- **The Space uses `sentence-transformers` for the encoder** instead
  of fastembed. Slightly larger Docker image, much simpler custom-model
  loading. The fine-tuned model loads from HF Hub on first request,
  ~2s cold-start cost.
- **`WINETONE_ENCODER` env var** lets you A/B against any
  HF-Hub-compatible sentence-transformer. Useful for measuring fine-tune
  lift on real queries.

## What's not in this release

For honesty: a few things we discussed and explicitly deferred.

- **Multi-layer (MLP) projection** for personalization — pushed back.
  The linear `A·L + b` works precisely because the identity prior is
  clean; MLPs lose that. With 5-13 labels per user, simple-plus-prior
  almost always beats deeper-plus-flexible.
- **Multi-modal embeddings (chemistry + text)** — researched in
  [docs/chemical-analysis-options.md](../chemical-analysis-options.md).
  IMS (the airport drug detector tech) is the right path; e-nose is
  the scrappy version. Not implemented yet — it's an architecture
  change, not a tweak.
- **Production-instance Clerk** — currently in development mode
  (small banner in the sign-in modal). Production instance requires
  CNAME for `accounts.tone.wine`, ~15 min of follow-up DNS work.

## How to try it

- Visit https://tone.wine
- Sign in with email / Google / GitHub
- Label 5-7 wines you know well, in your own words, with thumbs-up
  or thumbs-down per label
- Hit "Fit my taste profile"
- Ask for wines in your own vocabulary at `/ask`

If you want to run WineTone on your own infrastructure: clone the repo,
download [the release tarball](https://github.com/archisgore/WineTone/releases),
import via `winetone import-release`. The README's "Try it in five
minutes" path is unchanged.
